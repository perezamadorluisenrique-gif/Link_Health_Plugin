# Native Link Health — Agent Guide

## Overview

A WordPress plugin (v1.3.3, DB schema **2.3**) that scans posts/pages — and, opt-in, CPTs/media/comments/navigation menus — for broken links using `WP_HTML_Tag_Processor` and incremental WP Cron (5 posts every 15 min). Also includes an SEO audit module and a **Link Juice** module (internal link map + PageRank authority analysis). **Requires PHP 8.0+ and WordPress 6.2+.**

## Entry point

- `native-link-health.php` — defines constants (`NLH_VERSION` (1.3.3), `NLH_BATCH_SIZE=5`, `NLH_DB_VERSION=2.3`, plus `NLH_PLUGIN_FILE`/`BASENAME`/`DIR`/`URL`/`TEXT_DOMAIN`/`UPGRADE_URL`), loads all classes, boots the plugin. Also hooks `NLH_Activator::maybe_upgrade` on `admin_init` so existing installs get new schema (e.g. the v2.1 link-map tables, the v2.3 `source_type` column) without reactivation. Registers `delete_comment` → `nlh_cleanup_deleted_comment` alongside `deleted_post` cleanup.
- `nlh_bootstrap()` is called **immediately** (not on `plugins_loaded`). Hooks scanner to `nlh_run_batch` cron, `save_post`, and `deleted_post`. The `save_post` handler (`handle_post_saved`) temporarily unhooks itself during `update_post_link()` to avoid re-scan loops.
- Global function `nlh_update_post_link( $post_id, $old_url, $new_url ): bool` wraps `NLH_Scanner::update_post_link()`.

## File conventions

- Classes: `class-nlh-{name}.php`, class names `NLH_*`, methods `snake_case`, constants `NLH_*`.
- `includes/` — core logic (Scanner, Link Graph, Link Recommendations, Rules Engine, SEO Audit, Export, i18n, Activator, Deactivator).
- `admin/` — admin UI (Class, CSS, JS, templates for dashboard, SEO audit, settings).

## Tooling

- **No** `package.json`, `composer.json`, `Makefile`, CI config, or build step. Install manually: drop the folder into `wp-content/plugins/` and activate.
- **Tests** (since 1.3.0): `tests/test-scanner-helpers.php` is a standalone, dependency-free harness (run `php tests/test-scanner-helpers.php`) that reflection-tests the pure logic behind the no-false-positive engine and URL parser (`classify_error_type`, `trim_url_punctuation`, `parse_srcset`). Non-zero exit on failure. Network/WP-dependent paths still need the WP test suite — see `tests/README.md`.
- **Linter** (since 1.3.0): `phpcs.xml.dist` configures WordPress Coding Standards (text domain + `nlh`/`NLH` prefixes enforced; direct-`$wpdb` notices excluded by design). Needs `squizlabs/php_codesniffer` + `wp-coding-standards/wpcs` installed separately.
- Admin pages registered under **Tools** (`nlh-dashboard`, `nlh-seo-audit`, `nlh-link-juice`) and **Settings** (`nlh-settings`). All require `manage_options`.

## Key conventions

- **HTML parsing**: exclusively `WP_HTML_Tag_Processor`. Regex fallback only for bare URLs in text content.
- **DB tables**: 5 custom tables — `wp_nlh_link_errors`, `wp_nlh_link_events`, `wp_nlh_correction_log`, plus `wp_nlh_link_map` and `wp_nlh_link_scores` (Link Juice, DB v2.1). Direct `$wpdb` queries throughout, not `WP_List_Table`. **DB v2.3**: `wp_nlh_link_errors` gained `source_type varchar(20) DEFAULT 'post'` and its UNIQUE KEY widened to `url_hash_post_source (url_hash, post_id, source_type)` (`upgrade_to_2_3()`). For `source_type='comment'`, `post_id` stores the **comment ID** (enables per-comment cleanup + `get_edit_comment_link()`); for `menu`, `post_id=0`. `nlh_link_events`/`nlh_correction_log` are unchanged (no source_type), so comment/menu lifecycle events still key on `(url_hash, post_id)`.
- **HTTP**: HEAD requests with GET fallback when HEAD is refused/mishandled — `400`, `403`, `405`, `501`, or missing/`<100` status (servers/CDNs that block HEAD bot probes). 10s timeout (filterable via `nlh_http_timeout`). **5 redirection hops** (code value, despite readme saying 3). Default UA is a Chrome browser string, filterable via `nlh_user_agent`; browser-like `Accept`/`Accept-Language` headers are also sent (filterable via `nlh_http_headers`) so strict WAF/CDN sites don't return false 403s.
- **Conservative broken confirmation** (since 1.0.3): a **brand-new** failure (`WP_Error`, `>=400`, or `<100`) is NOT recorded immediately — it must fail `nlh_broken_confirm_threshold` (default **2**, filterable) consecutive probes, with a 1h backoff between probes (`nlh_fail_{hash}_{post}` counter, 7-day TTL). This prevents transient timeouts/DNS blips from creating false positives. **Exemptions**: URLs already on record (refreshed immediately) and forced manual scans (`$force=true`) record on the first probe so genuine breakage isn't delayed or hidden.
- **Internal links** (same host as the site) ARE scanned so broken internal links surface; only the home/root URL and WP system endpoints (`/wp-admin`, `/wp-login.php`, `/wp-cron.php`, `/xmlrpc.php`, `/wp-json`) are skipped. Disable loopback scanning via the `nlh_scan_internal_links` filter (returns bool). Note the SSRF guard still blocks `localhost`/private IPs, so internal scanning is effectively off on local dev installs.
- **Same-page deep links**: absolute URLs pointing back to the current post's own permalink (`https://site/this-post#section`) are routed to fragment validation, NOT fetched over HTTP (`is_same_page_fragment()`).
- **Soft status codes**: codes in `nlh_soft_status_codes` (default `[999]`) are treated as "could not verify" (back off + retry), not broken — like 429. Add `403`/`401` here to stop flagging bot-blocks globally. Honored by **both** the batch scanner AND `recheck_url()` (the manual re-check must not contradict the cron).
- **Bot-challenge detection** (since 1.3.2): `is_bot_challenge_response()` recognizes Cloudflare-style anti-bot interstitials (a `403`/`429`/`503` carrying `Cf-Mitigated: challenge`, or `Server: cloudflare` + a "Just a moment…"/`/cdn-cgi/challenge-platform` body marker — the HEAD→GET fallback supplies the body) and treats them as soft "could not verify", **not** broken. This kills the most common false positive (e.g. `w3.org`, `openai.com`) **without** softening genuine `403`s (e.g. a real `gunicorn` 403). Wired into the same gate in `check_and_record_url()` and `recheck_url()` as the soft codes; override via the `nlh_is_bot_challenge` filter.
- **Bare-URL extraction** strips `<script>/<style>/<code>/<pre>` blocks and HTML comments before regex, balances trailing `)`, and trims trailing sentence punctuation. Tag attributes scanned: `A/AREA[href]`, `LINK[href]`, `IMG/SCRIPT/SOURCE/VIDEO/AUDIO/IFRAME/EMBED[src]`, `OBJECT[data]`, `VIDEO[poster]`. `srcset`/`data-srcset` candidates on `IMG`/`SOURCE` are split out, and lazy-load attributes (`data-src`, `data-lazy-src`, `data-original`) on `IMG/SOURCE/IFRAME/VIDEO` are collected so deferred media URLs surface.
- **IDN support**: URLs with non-ASCII hostnames are converted via `idn_to_ascii()` before requests. `prepare_url_for_request()` guards against `preg_replace_callback('/u')` returning `null` on malformed UTF-8 (falls back to the original URL instead of emitting an empty request).
- **Caching**: Transients for ok links (48h), retry backoff (1h), suggestions (1h), SEO audit results (1 day), plus the `nlh_fail_` confirmation counter (7 days). `nlh_last_ok_` options track the last time a URL was confirmed ok. `clear_url_cache()` clears the ok transient, retry transients, last-ok option, AND the fail counter.
- **Fragment validation**: `#` links are checked against `id` attributes and `<a name>` anchors in the same post's content. The empty-anchors guard (skip validation when the post has no **explicit** anchors, since themes inject heading IDs at render) is evaluated on explicit `id`/`name` anchors ONLY; **heading slugs** (`sanitize_title()` of `<h1-6>` text) are then added as additional valid targets — they must never suppress the guard. `#top`, `#respond`, `#comments`, and `#comment-NNN` are always valid (browser/comment defaults rendered dynamically). Order matters: explicit anchors → guard → heading slugs → validate.
- **Fragment self-healing** (since 1.0.4): fragment errors (`status_code 0`, `error_message = 'Missing anchor fragment'`) are cleared once the anchor resolves. `reconcile_fragments()` deletes the record, logs a `fixed` event, and clears the retry transient for any candidate now in the post but no longer in the broken set. It runs both in `scan_posts()` (cron heals it) and inline via `heal_post_fragments()` on `save_post` update (no HTTP, so the author sees the fix immediately). It matches on the exact `error_message` literal so it never touches HTTP-error records (DNS/timeout also store `status_code 0`). Earlier versions only ever *inserted* fragment errors, so a corrected anchor stayed flagged forever.
- **Error-type classification** (since 1.0.3): `NLH_Scanner::classify_error_type( $status_code, $error_message )` returns `5xx / 4xx / fragment / dns / ssl / timeout` (timeout folds in connection errors). Used for the dashboard `data-error-type` filter AND the "Group by error type" view (grouped in PHP via `get_error_type_labels()`, not a SQL `CASE`). Replaced the old status-code-only bucketing that mislabeled every non-HTTP failure (DNS/SSL/fragment) as "timeout".
- **Cron**: 15-min `nlh_run_batch` action. Manual full scans run via AJAX (50 posts/chunk, offset-tracked). **Cron yields to manual scans** (since 1.0.4): `run_full_scan()` sets the `nlh_manual_scan_active` transient each chunk (5-min TTL, cleared on completion) and `run_batch()` returns early while it is set, so the two never probe the same URLs concurrently. The flag self-expires if a scan is abandoned. (The manual scan deliberately wins; cron is the interruptible side, so `run_full_scan`'s return shape is unchanged.)
- **Security**: `check_ajax_referer()`, `current_user_can('manage_options')`, `wp_safe_remote_*`, prepared statements in `$wpdb`.
- **SEO audit**: 5 checks — orphan pages, redirect chains, mixed content, invalid canonicals, redundant links. Results cached 1 day. **`redirect_chains` is now real (since 1.3.0)**: `trace_redirects()` follows each same-host internal link by hand (HEAD→GET fallback, `redirection=0`, loop-guarded, ≤6 hops) and flags only multi-hop chains (≥2 hops) or redirects that dead-end (≥400) — single clean `301→200` is intentionally NOT flagged (no-false-positives). Bounded to `nlh_redirect_chain_scan_limit` (default 50) URLs/run; only the site's own host is probed (no external hammering, no SSRF). *Detection is free*; the 301 auto-fix/management is the gated Pro feature (`nlh_pro_redirect_management_enabled`, extension point `nlh_seo_audit_after`).
- **Link Juice** (since 1.1.0): `class-nlh-link-graph.php` (`NLH_Link_Graph`) builds a persistent internal-link map (`wp_nlh_link_map`: source→target, internal/external) and computes a PageRank-style authority score per page (`wp_nlh_link_scores`). **Entirely offline** — content parse + `url_to_postid()` only, NO HTTP and none of the broken-link probe machinery (confirmation threshold / soft codes / SSRF). `url_to_postid()==0` is NOT treated as broken (archives/taxonomy/author/CPT legitimately resolve to 0); a 0 target is just an unresolved/external edge. The map is rebuilt per post as a side effect of `scan_posts()` and on `save_post` (`record_post()`); `deleted_post` drops a post as both source and target (`delete_post()`). Map extraction skips WP **system endpoints** (`/wp-admin`, `/wp-login.php`, `/wp-cron.php`, `/xmlrpc.php`, `/wp-json`, `*/feed/`) so e.g. a pasted wp-admin edit URL is not counted as a content link; internal resolution (`resolve_internal_target()`) tolerates an http/https scheme mismatch, and root-relative hrefs resolve against the site **origin** (not `home_url()`, which would double a subdirectory prefix — the same fix was applied to the scanner's `resolve_relative_url()`). When a post's link set actually changes, the scores are flagged stale (`nlh_juice_dirty` option, `mark_dirty()`/`is_dirty()`); the page shows a "Recalculate" banner and the flag clears on `compute_pagerank()` — so edits that haven't been re-scored don't look silently ignored. PageRank (damping `nlh_pagerank_damping` default 0.85, dangling-mass redistributed, ≤50 iters) recomputes on full-scan completion and via the manual **Recalculate** action (`rebuild_all()` = offline map rebuild + score). Admin page `nlh-link-juice` under Tools flags **orphans** (0 inbound), **dead ends** (inbound>0, 0 outbound internal) and **diluted** pages (outbound > `nlh_dilution_threshold`, default 100); supports re-pointing links (reuses `nlh_update_post_link` → `wp_update_post`, never direct SQL). The SEO-audit orphan check is intentionally left separate (it also counts nav-menu inbound links).
- **Link Juice UI** (`admin/js/nlh-juice.js`, `admin/partials/nlh-juice.php`): three layperson-friendly layers, all **vanilla SVG built with `createElementNS`/`textContent`** (no library, no `innerHTML` on post data — XSS-safe). (1) Per-page **flow diagram** in an expandable row tab — deterministic 3 columns *inbound → focal → outbound* with directional arrows (green in / blue out), neighbour nodes clickable to re-focus; caps at 10/side with "+N more". Data via `get_flow()` → `ajax_juice_details`. (2) **Site overview** — lazy-loaded, capped force-directed graph (`get_graph()` → `ajax_juice_graph`, top 150 by PageRank), node size/colour = authority/flag, wheel-zoom + drag-pan, click-to-focus dims non-neighbours and colours incident edges by direction. The hand-rolled force sim lives entirely in `simulate()`. (3) **Recommendations** (`class-nlh-link-recommendations.php`, `NLH_Link_Recommendations::get()`) — prioritized plain-language cards (trapped-authority hoarders > orphans > diluted) computed offline from map+scores; link suggestions use a **relevance cascade**: shared taxonomy term → shared title keyword (≥4 chars, EN/ES stopwords) → highest PageRank (so page-only sites with no taxonomy still get sensible candidates). Suggestions are **advisory/deep-link to the editor** (Tag_Processor can't safely insert/remove anchors — only re-point), so the UI never implies one-click structural edits.
- **Export**: CSV streams from `admin_post_nlh_export_csv` (nonce-guarded in `handle_export_csv()` via `check_admin_referer( 'nlh_export_csv_action', 'nlh_export_nonce' )`). Limited to 10k rows. Text cells (title, URL, error message) pass through `escape_csv_field()`, which prefixes values beginning with `= + - @ \t \r` with a single quote to neutralize **CSV/formula injection** (these fields derive from arbitrary post content). Keep that escaping if you add columns.

- **Extended scan scope** (since DB 2.3): the `nlh_scan_scope` option (`{post_types:[], comments:bool, menus:bool}`; Settings > Scan Settings via `render_scan_scope_field`/`sanitize_scan_scope`) opts into extra post types/media (merged into `get_scan_post_types()`), **comments** and **navigation menus**. `run_batch()` runs `scan_posts()` then `scan_comments()` then `scan_nav_menus()`. `scan_comments()` is cursor-paced (20 approved comments/tick via `nlh_comment_scan_last_id`, reset at end); `scan_nav_menus()` walks all menus each tick and skips internal links (`url_to_postid()>0`). Neither runs inside `run_full_scan()` (manual scans stay post-only).
- **Shared gated checker**: the no-false-positive pipeline is centralized in `NLH_Scanner::check_and_record_url($url, $post_id, $source_type, $force, $ignored, $rules_engine, &$counters)` — used by `scan_posts()`, `scan_comments()`, `scan_nav_menus()`. It owns the ok-cache/retry skips, soft/429 handling, the multi-probe confirmation gate, `discovered_at` preservation, regression/`fixed` events, and the rules engine (**post sources only**). Per-source state-key collisions are avoided by `state_key_suffix()` (posts keep the legacy `{post_id}` suffix; comments/menus get a `{source_type}_{post_id}` prefix), and the same applies to the 3-col UNIQUE KEY. `delete_stale_records()` takes a `$source_type` arg so removing a URL from one source never deletes another's records. Impact: menus = fixed 50, posts = `calculate_impact_score()`, comments = 0. Don't reintroduce the old inline post-only pipeline.

## Formerly-Pro UI (free/Pro split removed)

The free/Pro gating has been dismantled (as of 1.2.0). **Enabled/functional**: the Settings **Save** button (`submit_button` no longer disabled), the **Auto-fix Rules** editor (`nlh_auto_rules` — has a `name`, example moved to placeholder so an empty form is never saved as a live rule; the engine decodes a JSON-string option), and the **Scan Scope** field. CSV export, rules engine, and domain logic execute.

**The three backend-less stubs were UNREGISTERED in 1.2.0** (they previously rendered as dead "Available in Pro" inputs): **email notifications**, **scan frequency** (cron is fixed at `nlh_every_15_min`), and the **ignored-domains** textarea (the live ignore list is `nlh_ignored_urls`, managed from the dashboard). Their `add_settings_field()` calls were removed from `register_settings()`, but the `render_scan_frequency_field()` / `render_ignored_domains_field()` / `render_email_notifications_field()` methods are **kept (unregistered)** as starting points for the email/scheduling phases — don't re-register them until their backends exist. The false "CPT scanning is Pro" upsell footer in `nlh-dashboard.php` was also removed (CPT scanning is free via Scan Scope). `plugin_action_links()` still has a neutral "Upgrade to Pro" hook guarded by an empty `NLH_UPGRADE_URL` (hidden until a Pro product exists). `audit_redirect_chains()` is **no longer a stub** (see SEO audit above).

## Monetization & Pro gating (since 1.3.0)

The plugin is part of a multi-plugin suite. Licensing is **Freemius**, but it ships **disabled** so the free build shows no upsells and makes **no external calls**.

- **Constants** (in `native-link-health.php`, all `if(!defined)`-guarded so they can be overridden from `wp-config.php`): `NLH_FREEMIUS_ENABLED` (default **false**), `NLH_FREEMIUS_PRODUCT_ID` / `NLH_FREEMIUS_PUBLIC_KEY` (empty placeholders the owner fills), `NLH_AUTHOR_WP_HANDLE` (`[WP_ORG_USERNAME]`). Prices are **never** hardcoded — set externally in Freemius.
- **`NLH_Freemius::boot()`** (`includes/class-nlh-freemius.php`): no-op unless `NLH_FREEMIUS_ENABLED` is true AND the SDK exists at `vendor/freemius/start.php` AND credentials are set. Otherwise it shows an admin setup notice (never fatals). The real SDK is **not** committed — see `vendor/freemius/README.md`. When active it exposes `nlh_fs()` and feeds the Freemius pricing URL into `nlh_pro_upgrade_url`.
- **`NLH_Pro`** (`includes/class-nlh-pro.php`) is the single gate. `NLH_Pro::can('feature')` → `apply_filters("nlh_pro_{feature}_enabled", false)` (always false in core; the **separate Pro plugin overrides these — core never contains Pro code**). `NLH_Pro::is_enabled()` → `nlh_pro_ui_enabled` filter, defaulting to `NLH_FREEMIUS_ENABLED`. `NLH_Pro::upsell_card('feature')` renders an upsell **only** when `is_enabled()` and the feature isn't licensed — this is the one guard satisfying "never show disabled inputs". Gated feature keys: `bulk_fix`, `redirect_management`, `email_notifications`, `link_insertion`, `reporting`, `multisite` (Phase 5 order).
- **Extension points the Pro plugin hooks** (core stays advisory/detection-only): `do_action('nlh_link_broken_recorded', $url,$post_id,$source_type,$code,$error)` — fires once per newly-confirmed broken link (email/alerts); `do_action('nlh_dashboard_tools')` (bulk find-&-replace UI), `do_action('nlh_settings_after')` (email/scheduling/network settings), `do_action('nlh_seo_audit_after')` (301 manager), `do_action('nlh_recommendation_actions', $rec)` (one-click link insertion), filter `nlh_pro_upgrade_url`.
- **The detection engine is NEVER gated** — the no-false-positive pipeline is the brand and stays free. Only automation/scale/monitoring/agency tooling is monetizable.
- **Health-score history** (free data trail for the Pro trends/report feature): `NLH_Link_Graph::record_health_snapshot()` appends one `nlh_health_history` option entry (date→score, capped ~90 days) on every `compute_pagerank()`; read via `get_health_history()`. Cleaned in `uninstall.php`.

## Dashboard health overview & speed (since 1.3.0)

- **Unified Link Health Score**: `NLH_Admin::render_health_overview()` renders the dashboard cover — one headline blending broken-link detection (60%) with the authority `calculate_health_score()` (40%), a band label, and drill-down cards into broken links / authority / orphans / SEO. Falls back to detection-only when the graph was never computed.
- **Prioritization**: `get_batch_posts()` orders never-scanned posts by `modified DESC` (fresh/just-edited content first), then fills remaining slots with the stalest re-checks.
- **First-value UX**: per-post "Scan Now" meta box (`scan_post_now()` + `nlh_scan_post` AJAX), a real ETA on the full-scan progress bar (`nlh-admin.js`), and a "gentle, 100% local, no cloud" positioning note on the dashboard.

## Translation

POT file at `languages/native-link-health.pot` (318 msgid entries, no plurals; regenerate with `wp i18n make-pot` after changing any string). A complete **Spanish (es_ES)** translation ships: `languages/native-link-health-es_ES.po` (+ compiled `.mo`). Regenerate the `.mo` after editing the `.po` with `wp i18n make-mo languages/native-link-health-es_ES.po`. Keep all three in sync when strings change. Text domain: `native-link-health`. (A handful of placeholder strings still lack `/* translators: */` comments — `make-pot` warns about them.)
